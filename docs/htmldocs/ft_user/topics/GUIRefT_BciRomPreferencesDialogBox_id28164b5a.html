BCI-ROM Preferences Dialog Box

To access: The dialog box is opened when exporting BCI-ROM files.

Use this dialog box to specify the format of the BCI-ROM files that you are exporting, request an acceptable relative error value, and to set a global HTC range.

Description

BCI-ROM Format

Use this dropdown list to choose from:

  • Matrices - Files exported in this format are compatible with mathematical programing tools such as MATLAB® and GNU Octave. Files of this type can also be solved with the BCI-ROM Matrices solver included with Simcenter Flotherm.

    Note:

    This is the default option.

    During the export a Matrices Files.zip file is created.

  • FMU (Co-Simulation) - Files exported in this format can be solved in FMI compliant tools such as Simcenter Flomaster™, Simcenter AMESim™, and Ansys Simplorer®.

    During the export an FMU (Co-Simulation) Files.zip file is created.

  • VHDL-AMS - Files exported in this format can be solved in tools such as SystemVision® Cloud, Xpedition®.

    During the export a VHDL-AMS Files.zip file is created.

Acceptable Relative Error

The requested acceptable relative error. The default value of 0.001 is recommended for most applications.

Larger values (up to 0.01) can be used to accelerate the extraction process.

Smaller values (down to 0.0001) can be used when accuracy is paramount, however, use with caution, as this will impact on the calculation time required for extraction.

Minimum HTC

Sets the HTC range that is exercised for all Ambient attributes attached to the model, either through the solution domain faces or through cutouts.

The default range of 1-to-10,000 W/m2K is used for Delphi model generation in FloTHERM PACK, and is well suited for package level applications.

The range can be customized for any application, for example, larger values can be used for cold plate type boundary conditions.
