FloXML Files

FloXML files are XML files that can be imported into Simcenter Flotherm.

FloXML files must conform to the FloXML Schema. There are two FloXML file types:

Project FloXML files

which contain geometry and solution domain information. Project files must contain a minimum FloXML project definition.

Geometry FloXML files

which only contain geometry information.

The name defined within the XML file, for example:

- <xml_case> 
    <name>Heatsink</name> 
    . 
    . 
    . 
</xml_case>

is used to name the Simcenter Flotherm project/assembly that is created from the file.

An alternative to importing an XML file directly using the GUI, is to convert the XML file to PDML format using the flogate_cl utility, for example:

flogate_cl -i XML -r my_example.xml -o PDML -w my_example.pdml

See “Converting Files to PDML Format”. The resultant PDML file can then be imported into Simcenter Flotherm.

Minimum Project FloXML Definition

A valid project FloXML file must contain a minimum set of default definitions.

An example set of default definitions is supplied within file Default.xml in the following location:

install_dir\flosuite_vversion\flotherm\examples\FloXML\FloXML Files\Project FloXML Examples

FloXML Schema

The schema *.xsd files are in the following location:

install_dir\flosuite_vversion\common\WinXP\lib\flotherm\PlugIns\XML_PLUGIN_SCHEMA

Note:

The FloXML Schema supports the definition of all objects and settings that can be made through the GUI (objects, attributes, grid settings, model settings, solver settings, and so on) except auxiliary variables and Power Maps.

FloXML File Examples

Examples of FloXML files are provided in the installation:

install_dir\flosuite_vversion\flotherm\examples\FloXML\FloXML Files

The examples, which are grouped as Assembly or Project FloXML files, are described in Table 1 and Table 2.

Table 1. Assembly FloXML Examples

Filename

Description

2R-Model.xml

A two-resistor model, based on a network assembly and three network nodes (Junction, Case, and PCB). The junction node has a power output of 3.2 W. The junction-to-board resistance is 22.2 K/W, and the junction-to case-resistance is 1.2 K/W. A monitor point is attached to the junction.

Advanced-Resistance.xml

A resistance object and advanced resistance attribute with loss coefficients based on device velocity.

Block.xml

A 20 mm × 10 mm × 1 mm aluminum block.

Nested-Assemblies.xml

Three levels of nested assemblies, the innermost assembly containing a cuboid.
Table 2. Project FloXML Examples

Filename

Description

All-Objects-Attributes-Settings-FullModel.xml

A root assembly containing a set of different objects with attributes to illustrate a wide variety of XML definitions.

Default.xml

Not a project, but the minimum definitions required by valid Project FloXML files.

Heatsink-Windtunnel-FullModel.xml

A 10-folded-fin heat sink on a 40 mm × 40 mm base with a 1 W heat source, and a 0.001 m3/s fixed volume flow in the fin direction.

PDML-Referencing-FullModel.xml

An example of an project XML file that includes a reference to a PDML file. It is assumed that the PDML file is named PCB.pdml and is located in the C:\ folder:


<pdml>
<name>PDML</name>
<file>c:\PCB.pdml</file>
<position><x>0</x><y>0</y><z>0</z></position>
