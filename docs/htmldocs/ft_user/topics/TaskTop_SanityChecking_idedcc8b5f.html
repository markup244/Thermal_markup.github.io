Sanity Checking

As the model is refined, it is good practice to regularly run sanity checks before running the solver.

Procedure

Choose Solve > Sanity Check.

Results

Any Error, Warning, or Information messages are output to the Message Window, followed by a I/9031 message that gives total numbers of the three different message types. Error messages stop the solver from running and therefore must be addressed.
