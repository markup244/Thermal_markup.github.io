Grid Space Overlap When Localizing

When localized grid spaces overlap, the smallest grid cell specification is used to define the overlapped space.

This is shown in Figure 1. Note that the application of grid constraints for the overlapped grid spaces is independent of the hierarchical positions of the objects in the data tree, it is always based on the smallest grid size.

Handling of Legacy Overlapped Localized Grids

Prior to V11.0, overlapping of two grid spaces was not permitted. The localization settings for an object lower in the data tree hierarchy was ignored if the spaces overlapped.

How these projects are handled after importing into V11.0 and higher depends on whether or not results are included:

  • If results are NOT included (for example, by importing PDML) then the grid is updated, allowing overlapping.

  • If results are included (for example, by importing a PACK file with Results), then the legacy grid is retained to be compatible with the results.
